## About This Project

A single-page surgical revision app (disease compendium + quiz) for exam
study. It's a **prototype that is live** at
<https://mos-surgery-v1-cd.netlify.app>.

The entire app ships as **one file: [index.html](index.html)**. There is no
server. Source lives in the local `.jsx` files, then [build.js](build.js)
precompiles JSX into plain JavaScript and writes the deployable `index.html`.

---

## Tech Stack

| Layer    | Tool                                   | Notes                                                            |
| -------- | -------------------------------------- | ---------------------------------------------------------------- |
| UI       | React 18.3.1 + ReactDOM (UMD)          | Loaded from `unpkg.com` CDN with SRI `integrity` hashes          |
| JSX      | `esbuild` 0.28.0                       | Invoked by [build.js](build.js); compiles JSX **before deploy**  |
| Styling  | Plain CSS                              | Authored in [styles.css](styles.css), inlined into `index.html`  |
| Data     | Hand-written JS arrays                 | Built by [combine.js](combine.js) into [data.jsx](data.jsx)      |
| Deploy   | Netlify (manual CLI deploy)            | Site `mos-surgery-v1-cd`, id `8bcc9aeb-87f0-4b95-911e-befb56c9a126` |

> **Internet required.** Because React comes from the CDN, the page is **blank
> when offline**. That is not a bug in our code.

## Common Commands

```bash
# Rebuild data.jsx from the per-department tmp_*.js files
node combine.js

# Rebuild the deployable single-file app after source edits
node combine.js && node build.js

# Deploy the current files to the LIVE site (production)
netlify deploy --prod --dir .

# Who am I / which site is linked
netlify status

# Open the app locally in a browser
open index.html
```

**Easiest path:** instead of memorising the deploy command, just run the
**`/deploy`** skill — it deploys and then verifies the live page actually
renders. See `.claude/skills/deploy/`.

## ⚠️ Deploy after EVERY change (this site is manual-deploy)

This project is **not** connected to git auto-deploy. Editing a file locally
changes **nothing** on the live site until you explicitly deploy. A change is
not "done" until it is live.

**The rule:** after any edit to `index.html` (or any source file that feeds
it), immediately run `/deploy` (or `netlify deploy --prod --dir .`). Never
leave the working tree ahead of production.

To verify a deploy worked, confirm the live page renders (not just that the
HTTP request returns 200 — a crashed React app still returns 200 with a blank
body). The `/deploy` skill does this check for you.

## Project Structure

```
surgery/
├── index.html          ← THE deployable file. Generated by build.js.
├── styles.css          ← stylesheet (a copy is inlined into index.html)
├── combine.js          ← build script: merges tmp_*.js → data.jsx
├── data.jsx            ← GENERATED by combine.js (do not hand-edit the data)
├── tmp_*.js            ← source disease data, one file per department (+ tmp_quiz.js)
├── app.jsx             ← source for the App block in index.html
├── components.jsx      ← source for the shared-components block
├── views.jsx           ← source for the views block
├── tweaks-panel.jsx    ← source for the tweaks/edit-mode block
├── netlify.toml        ← publish = ".", Cache-Control: no-store
└── .netlify/           ← Netlify link state (siteId)
```

`index.html` is assembled from those source `.jsx` files by [build.js](build.js)
in this order: **tweaks-panel → data → components → views → app**. When you
change a source `.jsx`, run `node build.js` so the deployable file is updated.
When disease data changes, run `node combine.js && node build.js`.

## Notes & Gotchas

- **`window.SK_DATA` is mandatory.** The UI reads all generated surgical data
  from `window.SK_DATA`. The data file **must** end with:

  ```js
  window.SK_DATA = { DEPARTMENTS, DISEASES, PROCEDURES, IMAGE_QUIZ_BANK, ALGORITHMS, SCORING, QUIZ_BANK };
  ```

  If this line is missing, the app crashes on first render with
  *"Cannot read properties of undefined (reading 'DEPARTMENTS')"* and the page
  is **blank**. `combine.js` now writes this line into `data.jsx` automatically
  — keep it that way. `build.js` then compiles it into `index.html`.

- **Don't hand-edit data in `data.jsx`.** It's generated. Edit the relevant
  `tmp_*.js` file, then run `node combine.js && node build.js`.

- **Performance mode is build-time JSX.** `index.html` should include
  `<meta name="build-mode" content="precompiled-jsx">`, React production UMD
  scripts, and a normal `<script>` tag. It should **not** include
  `@babel/standalone` or `<script type="text/babel">`.

- **Desktop layout only.** The viewport is fixed at `width=1280` — this app is
  not mobile-responsive by design.

- **Diagnosing a blank page** (in order): (1) open DevTools console and look for
  a red error — a thrown error during render blanks the page; (2) check you're
  online (CDN scripts); (3) verify the `window.SK_DATA = {…}` line is present;
  (4) confirm the live site is actually the latest deploy (`netlify status`,
  compare with your local file).
